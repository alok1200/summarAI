   176→Stage Summary:
   177→- Long videos (up to 50 hours) now work end-to-end. The pipeline is:
   178→  1. Chunk: ~22K-char chunks at segment boundaries (preserves timestamps)
   179→  2. Map: parallel per-chunk LLM calls (4-way concurrency) with per-chunk error isolation
   180→  3. Reduce: single LLM call that merges per-chunk results into a coherent final answer
   181→  4. Stream: progress indicators + final answer stream to the client in real time
   182→- Speed: parallel processing gives ~4x speedup over sequential. A 73K-char transcript summary completes in 15s (vs 60+ seconds sequential). First byte always <1s, so no proxy 502s.
   183→- Quality: for interview Q&A, the reduce step is explicitly told to pick DIVERSE questions spanning all chunks (not cluster around one segment). Test confirms questions come from timestamps across all 4 chunks.
   184→- Q&A mode: retrieval-augmented. For each user question, an LLM picks the top 3 most relevant chunks (from the topic index), and only those chunks are injected into the context. Off-topic questions short-circuit instantly with the canned reply, no main LLM call needed.
